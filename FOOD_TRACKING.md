# 📸 Подсчет калорий по фото - Руководство

## Описание функции

Этап 2 проекта CalorAI реализует анализ фото еды с помощью Google Gemini AI для автоматического подсчета калорий и БЖУ.

## 🎯 Основные возможности

### 1. Анализ фото
- Распознавание продуктов на изображении
- Оценка веса каждого продукта в граммах
- Автоматический расчет калорий и БЖУ (белки, жиры, углеводы)
- Уровень уверенности распознавания для каждого продукта

### 2. Подтверждение приема пищи
После анализа фото пользователь получает:
- Список распознанных продуктов с весом
- Калории и БЖУ для каждого продукта
- Общую калорийность приема пищи
- Три варианта действий:
  - ✅ **Да, добавить** - подтвердить и добавить в дневной баланс
  - ✏️ **Изменить** - отредактировать вес продуктов или удалить продукты
  - ❌ **Отменить** - не добавлять прием пищи

### 3. Редактирование
Пользователь может:
- Изменить вес любого продукта (пересчет КБЖУ происходит автоматически)
- Удалить продукт из списка
- Вернуться к экрану подтверждения после изменений

### 4. Дневная статистика
После подтверждения пользователь видит:
- Дневную норму калорий
- Съеденные калории
- Оставшиеся калории

## 🚀 Как использовать

### Настройка

1. **Получите API ключ Gemini:**
   - Перейдите на [Google AI Studio](https://makersuite.google.com/app/apikey)
   - Создайте новый API ключ
   - Скопируйте его

2. **Добавьте ключ в `.env` файл:**
   ```env
   GEMINI_API_KEY=your_gemini_api_key_here
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

### Использование в боте

1. **Завершите онбординг** (если еще не сделали)
   - Используйте команду `/start`
   - Заполните все данные (пол, возраст, рост, вес, активность)

2. **Отправьте фото еды** в бот
   - Сфотографируйте вашу еду
   - Отправьте фото боту
   - Подождите анализа (обычно 2-5 секунд)

3. **Проверьте результат**
   - Бот покажет распознанные продукты
   - Проверьте вес и КБЖУ каждого продукта
   - При необходимости отредактируйте

4. **Подтвердите или отмените**
   - Нажмите ✅ "Да, добавить" для подтверждения
   - Или ❌ "Отменить" для отказа

## 📊 Структура данных

### База данных

#### Таблица `meals` (приемы пищи)
- `meal_id` - ID приема пищи
- `user_id` - ID пользователя
- `meal_date` - дата приема пищи
- `status` - статус ('draft' или 'confirmed')
- `total_calories` - общие калории
- `total_proteins` - общие белки
- `total_fats` - общие жиры
- `total_carbs` - общие углеводы
- `photo_file_id` - ID фото в Telegram

#### Таблица `meal_items` (продукты в приемах)
- `item_id` - ID продукта
- `meal_id` - ID приема пищи
- `product_name` - название продукта
- `weight` - вес в граммах
- `calories` - калории
- `proteins` - белки в граммах
- `fats` - жиры в граммах
- `carbs` - углеводы в граммах
- `confidence` - уровень уверенности (0.0-1.0)

### API Gemini

Для анализа используется модель `gemini-1.5-flash`, которая:
- Распознает продукты на изображении
- Оценивает вес на основе визуального анализа
- Рассчитывает КБЖУ из базы питательных данных
- Возвращает уровень уверенности для каждого продукта

## ⚠️ Важные замечания

### Точность распознавания
- Gemini API дает приблизительные результаты (±10-20%)
- Вес продуктов оценивается визуально
- Всегда проверяйте и корректируйте данные при необходимости
- Продукты с низкой уверенностью (<0.5) помечаются предупреждением

### Ограничения
- Распознаются только **видимые** продукты
- Скрытые ингредиенты не определяются
- Для сложных блюд может потребоваться корректировка
- Качество фото влияет на точность распознавания

### Рекомендации для лучших результатов
1. Делайте фото при хорошем освещении
2. Располагайте еду на контрастном фоне
3. Снимайте сверху для лучшего обзора
4. Избегайте размытых фотографий
5. Показывайте продукты целиком, без обрезки

## 🔧 Техническая информация

### Основные компоненты

1. **FoodAnalyzer** (`tgbot/services/food_analyzer.py`)
   - Взаимодействие с Gemini API
   - Парсинг и валидация ответов
   - Пересчет КБЖУ при изменении веса

2. **MealModel** (`tgbot/models/meal.py`)
   - CRUD операции для приемов пищи
   - Управление статусами (draft/confirmed)
   - Расчет дневной статистики

3. **FoodTrackingStates** (`tgbot/misc/states.py`)
   - FSM состояния для отслеживания питания
   - waiting_for_photo
   - confirming_meal
   - editing_product
   - editing_weight

4. **food_tracking_router** (`tgbot/handlers/food_tracking.py`)
   - Обработка фото
   - Подтверждение/отмена приемов
   - Редактирование продуктов

### Workflow

```
Пользователь отправляет фото
         ↓
FoodAnalyzer анализирует через Gemini API
         ↓
Создается meal со статусом 'draft'
         ↓
Добавляются meal_items
         ↓
Показывается результат с кнопками
         ↓
    Пользователь выбирает:
    ├─ Подтвердить → status='confirmed', обновляется статистика
    ├─ Изменить → редактирование веса/удаление продуктов
    └─ Отменить → удаление meal
```
